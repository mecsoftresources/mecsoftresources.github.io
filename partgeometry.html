Part Geometry

Part Geometry is the CAD design geometry that exists in the part file. This design data could consist of both 2D and 3D data.  Part geometry is utilized in the computation of toolpaths in the  module.  Geometry is used in utilized differently in each of the machining operations in MILL.  This is summarized below:

Part Geometry Type by Configuration

2½ Axis Operations:  Typically uses 2D wireframe geometry.  3D wireframe geometry is used in some operations and the 3D surface and/or mesh data as well as the 3D features can be optionally used for certain computations.

3 Axis Operations:  Uses 3D surface and or mesh data of the entire visible part.  The cutter can be contained using 2D wireframe data.  In some methods 3D wireframe data could be also utilized.

 

 

 

3 Axis Operations:  Uses 3D surface and or mesh data of the entire visible part.  The cutter can be contained using 2D wireframe data.  In some methods 3D wireframe data could be also utilized.

 

 

4 Axis Operations:  Similar to 3 Axis operations.

 

 

 

 

5 Axis Operations:  These operations require 3D NURBS surfaces to exist.  5 Axis operations cannot work with mesh geometry.  The surfaces to machine are selected in the operations.  If surfaces are not selected they will not be considered in the toolpath computations.

Part Geometry constitutes the end product of the manufacturing operation. This is also the design model. Design models in various data formats can be imported into Rhino. These design models can either be solid models or surface models or even faceted triangle models.
