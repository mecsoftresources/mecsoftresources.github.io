My Post is Missing! It is not in the Current Post-Processor List!

The selection of current post processors is displayed in the Set Post-Processor Options dialog shown below.  You must select a Post from the Current Post-Processor dropdown menu.  If for some reason you cannot locate your post in the menu list, it means one of two things:

1.Your post definition file has been moved and no longer resides in the folder where the Set Post-Processor Options dialog is looking.

2.The Set Post-Processor Options dialog is looking in the wrong folder for your post.

Either way, you need to keep track of where your post-processor definition file(s) are located.  These post files are very valuable to you.  Over time, you and/or MecSoft support has customized these post definition files to run specifically for your CNC machine controller.  The list below provides some guidance on handling the post definition files.

 

Guidance on handling the post definition files:

•Always Know Where Your Post Files Are!
Since these post definition files are specific to your CNC machine, they transcend any MecSoft plugin version.  This means that your post definition files and not version-specific.  Always know where your post definition files are stored.  Many users simply keep them in the My PC/Desktop/ folder so that they are always visible on your screen desktop.  However, even this has risks.  What if your PC crashes?  The best method is to store these files on a company network drive or on a cloud drive.

•Do NOT Store Post Definition Files in Your Install Path!
If we or you have modified your post definition files, it is important that you DO NOT store these modified post definition files in the default plugin install folder.  Why?  Because ALL of the files in your install path are subject to being overwritten during the next plugin update!  

•When Editing a Post Definition:
If you are familiar with the Post-Processor Generator, you understand that you can edit your post definition files. IF YOU EDIT A POST DEFINITION FILE, ALWAYS USE SAVE AS AND NOT SAVE .  Use Save As and name the post differently.  For example if you use the Fadal.spm post definition file and you have edited it, save it as a new name such as Fadal1.spm to indicate the revision.

•When renaming a post definition file:
Always keep the file name to 15 characters or less.  This is the limit number of characters that will display in the Set Post-Processor Options dialog.  You can rename the files using more than 15 characters and you will be able to read the file name in the Windows File Browser, but not when it is in the Set Post-Processor Options dialog.
Tips to Locating Your Post Definition Files

Your post definition files will always the file extension of *.spm, such as Fadal-REV1.spm

If you cannot locate your post, perform a Windows search from a file browser window and search for *.spm.  The search results will span the current folder location - meaning that all files in the current folder are searched.  To search your entire pc, navigate to My PC and enter *.spm in the search field as shown below.

Windows File Browser Search Bar
How to Let MecSoft CAM Know Where Your Post Files are Located

In order for your CAM plugin to know where your post definition files are located, you need to make sure the Set Post-Processor Options dialog is looking in the correct folder.

1.From MecSoft CAM, go to the Machining Browser and select the Program tab.

2.Select the Post icon to display the Set Post-Processor Options dialog.

The Post icon

3.From the Set Post-Processor Options dialog, make sure Use Legacy Post is selected, then move down to where it says "Folder where post-processor files are located:

The Set Post-Processor Options dialog

4.The current folder path is shown in the dialog.  If this path is incorrect, select the button to display the Browse for Folder dialog.

Browser for Folder dialog

5.Use this dialog to locate the folder where you have placed your post definition files and then pick OK.

6.Now pick OK from the Set Post-Processor Options dialog.

7.Now repeat steps 1-3 above and you should see your post definition listed in the Current Post-Processor selection menu.
