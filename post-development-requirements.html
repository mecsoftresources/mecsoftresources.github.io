Post Development Requirements

Need a new post processor developed for your CNC machine?  Locate the CNC machine type listed in one of the sections below and then supply us with the required information.  We will then make a determination of post feasibility.

 

Required to determine the feasibility of writing a post for a 3 axis machine

1.G & M code specifications from the controller manual

2.Information on canned cycles for drilling, tapping, boring operations, (G81 - G89)

3.Arc output formats for Circular interpolations (G02/G03),

4.Start and End Block information

5.Tool Change definition

6.A sample G code output that runs on your controller.
Required to determine the feasibility of writing a post for a 4 axis machine

1.4 axis - Rotary table or Head

2.Orientation of rotary axis - Parallel to X or Y and Rotation axis code for rotary axis (for example, A for rotation about X axis)

3.G & M code specifications from the controller manual

4.Information on canned cycles for drilling, tapping, boring operations, (G81 - G89)

5.Arc output formats for Circular interpolations (G02/G03),

6.Start and End Block information

7.Tool Change definition

8.A sample G code output that runs on your controller.
Required to determine the feasibility of writing a post for a 5 axis machine

1.5 axis machine configuration (Head-Head / Table-Head/ Table-Table) and Machine kinematics

2.Rotation direction, angle limits for primary & secondary axis.

3.Are the primary and secondary axis orthogonal?

4.Does the machine use right hand coordinate system?

5.Information on Primary and Secondary axis. (For example: On a head-head configuration, the Primary axis rotates about Z axis and the Rotation axis code is represented by the as "C' and secondary axis rotates about Y axis, the rotation axis code is represented as "B")

6.G & M code specifications from the controller manual

7.Start and End Block information

8.Tool Change definition

9.Arc output formats for Circular interpolations (G02/G03),

10.A sample g-code that has 5th axis output.

11.How are drilling cycles handled in 5 Axis. Does the machine expect simulated cycles (G01) or canned cycles (G81, G83 etc..)

12.Does the machine/controller expect the coordinate values to be output in rotated coordinates or in world coordinates?

13.For machines that use rotary head, does the machine compensate for gage length or does it expect the CAM software to compensate for it?

14.What type of application will the 5 axis machine being used for? Do you have any sample files that can be provided so we could take a look at it to see if our CAM product can support programming these parts?
Required to determine the feasibility of writing a post for a 2 axis laser CNC machine

1.G & M code specifications from the controller manual

2.Arc output formats for Circular interpolations (G02/G03),

3.Start and End Block information

4.A sample G code output that runs on your CNC.

5.Codes used to turn off the laser before a transfer / rapid motion and turn on the laser before start of a cut motion.
Required to determine the feasibility of writing a post for a 2 axis plasma CNC machine

1.G & M code specifications from the controller manual

2.Arc output formats for Circular interpolations (G02/G03),

3.Start and End Block information

4.Codes used to turn off the plasma torch before a transfer / rapid motion and turn on the plasma torch before start of a cut motion.

5.A sample G code output that runs on your CNC.
Required to determine the feasibility of writing a post for a 2 Axis TURN machine

1.G & M code specifications from the controller manual

2.Start and End Block information

3.Tool Change definition

4.A sample G code output that runs on your controller.

5.Threading cycles supported with cycle parameters and sample output.

6.Arc output formats for Circular interpolations (G02/G03).

7.Note: TURN only supports 2 axis (X and Z) programming. Live tooling is not supported.
